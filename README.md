# Composio App Research Case Study

This repo contains a small research pipeline that classifies 100 apps by auth, self-serve access, API surface, and buildability, then renders a single-page HTML case study.

## Run locally

1. Create / activate the provided Python environment.
2. Run the research generator:

```powershell
c:\Users\VIVIAN\Desktop\Composio\.venv\Scripts\python.exe research\generate_report.py
```

3. Preview locally with a static server:

```powershell
c:\Users\VIVIAN\Desktop\Composio\.venv\Scripts\python.exe -m http.server 8000 -d c:\Users\VIVIAN\Desktop\Composio
```

4. Open `http://localhost:8000/dist/index.html` in a browser.

## Host on GitHub Pages

1. Create a new GitHub repository and push this folder to the `main` branch.
2. In GitHub, go to `Settings` > `Pages`.
3. Set `Build and deployment` to `GitHub Actions`.
4. Push to `main` again, or open the `Actions` tab and run the `Deploy GitHub Pages` workflow manually.
5. After the workflow succeeds, GitHub Pages will show the live site URL in the workflow run and in `Settings` > `Pages`.

## Generated outputs

- `dist/index.html` - final case study page
- `data/research.json` - structured research output
- `data/verification.json` - verification sample and result notes
- `data/sources.json` - evidence links used by the generator

These files are regenerated by `research/generate_report.py` and ignored by git so the repo stays clean.

## Notes

The page is designed to be self-explanatory in under two minutes and includes the workflow, the findings, and the verification sample.
